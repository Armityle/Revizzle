Ten steps for a HTTPS handshake:
  1. Client and server build a TCP connection
  2. Client sends specifications to the server (SSL/TLS version, compression method, ciphersuites)
  3. Server selects the highest supported version of SSL/TLS version, compression method, ciphersuites
  4. Server sends certificate
  5. Certificate must be trusted by the client or someone they trust.
  6. A key might be exchanged depending on the ciphersuite.
  7. Client and server set up a key for symmetric encryption.
  8. Client requests all communication be encrypted.
  9. Server verifies the Message Authentication Code and the message can be decrypted.
  10. Handshake is complete. A secure network is now established.
